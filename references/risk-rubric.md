# CodeFlow Guard 风险规则

## P0 阻断合并

- 删除或绕过认证、鉴权、权限校验
- 可能造成数据丢失、资金错误、隐私泄露或生产不可用
- 数据库变更不可逆且没有迁移或回滚说明
- 硬编码密钥、令牌、密码或敏感连接串

## P1 高风险

- 修改登录、支付、订单、权限、文件上传、回调等关键链路
- 修改公开 API、数据库 schema、缓存 key、消息队列协议
- 引入新依赖、升级核心依赖或修改构建/部署配置
- 缺少关键路径测试或回归验证

## P2 中风险

- 修改业务逻辑但测试覆盖不足
- 缺少边界条件、异常分支或空值处理
- 前端表单、路由、状态管理改动可能影响用户流程
- 性能、并发或资源释放存在潜在问题

## P3 低风险

- 命名、可读性、重复逻辑、日志质量或小范围可维护性问题
- 文档、注释或错误提示可改进
